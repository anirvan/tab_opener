1 tab\_opener
2 ==========
3 
4 tab\_opener is an aplication that focuses on the thing that you do most of the
5 time in a file manager: open files. It is aimed at the developer type of person
6 who want's to quickly navigate projects and cares a great deal about optimizing
7 the workflow for a hand full of filetypes.

21 Features on a glace
22 -------------------
23 
24 * two configurable primary actions for filetypes
25 
26  * the idea is that you can access two opening modi for a filetype instead of
27  only one like in traditional file managers and hunt for the other in the
28  context menu
29 
30 * stays out of your face when you don't need it
31 
32  * each time you open a file with tab\_opener, it closes it's self. This way it
33  is not cluttering your alt-tab / currently running application list. It's a
34  light weight application, so you can quickly start it again the next time
35  you need it. No more 5 file browsers hanging around.
36 
37 * quickly execute commands without the need for a separate terminal
38 
39  * execute commands in the currently selected path. show stdout, stderr
40  (highlighted) and return code
41 
42 * quick access to recently used files and commands
43 
44  * the 10 recently used files and comannds are just one click away
45 
46 * quick access to most used files and commands
47 
48  * same as with recently used files and commands, just for usage count
49 
50 * file specific notes (work in progress)
51 
52  * currrently the notes widget is just one static wall, but the next feature
53  will be file type specific notes
54 
55 * sqlite database backend
56 
57  * in case you want to save / edit the saved data, you can do it with a sqlite
58  editor. no obsure binary blob save files. the application database file can
59  be found in the home folder and is called .tab_opener.db
